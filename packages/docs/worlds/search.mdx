Before an agent can reason about the facts in a world, it must find the correct
starting point. **Hybrid search** acts as a powerful disambiguation tool,
allowing the agent to evaluate a natural language query and instantly resolve it
to the correct item (IRI).

Rather than relying on a single vector index, Worlds fuses three complementary
signals to guarantee retrieval accuracy.

## Signals

| Signal            | Technique                   | What it captures         |
| :---------------- | :-------------------------- | :----------------------- |
| **Semantic**      | Vector embeddings, 1536-dim | Conceptual meaning       |
| **Keyword**       | FTS5 / BM25                 | Exact term matches       |
| **Graph context** | RDF relationship filters    | Structural relationships |

## Reciprocal rank fusion

Results from each signal are merged using **Reciprocal Rank Fusion (RRF)**, an
industry-standard algorithm that produces a single unified ranking:

$$score = \sum_{d \in D} \frac{1}{60 + rank(d)}$$

This ensures that a result ranked highly by _multiple_ signals surfaces to the
top, while noise from any single signal is suppressed.

## Why not only vector search?

Standard vector search excels at finding semantically similar text, but agents
require factual precision. If an agent searches for "Ethan's manager", a pure
vector query might return "Gregory's manager" simply because the sentence
structures are nearly identical in vector space.

By layering keyword precision (BM25) and structural graph filters on top of the
vector embeddings, hybrid search guarantees the agent disambiguates the exact
item it needs before executing strict symbolic logic.
