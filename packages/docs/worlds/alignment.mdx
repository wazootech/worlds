title: "Update and alignment"

Worlds is a malleable context engine. To maintain a living world, agents must 
continuously update the state through verifiable mutations and align those 
updates with human and autonomous preferences.

## Updating state

A world mutates its graph state using **RDF patches**—granular additions and 
deletions of facts. Because Worlds functions as a chronological, append-only 
ledger, these patches preserve historical truth and temporal evolution.

### SPARQL updates

The most common way to mutate state is executing SPARQL updates. When an agent 
runs an `INSERT` or `DELETE` command, Worlds translates it into a deterministic 
patch.

```sparql SPARQL
PREFIX corp: <http://corp.example/>

INSERT DATA {
  <corp:acme> corp:ownedBy <corp:parent> .
}
```

### Ingestion strategies

*   **One-time import**: Bootstrapping a world via CLI or SDK using existing 
    Turtle or N-Quads files.
*   **Direct SDK ingestion**: Real-time insertion from agentic loops using 
    [`@wazoo/worlds-sdk`](https://jsr.io/@wazoo/worlds-sdk).
*   **Data synchronization**: Hooking into external event streams via a 
    forward-sync proxy to translate webhooks into graph facts.

## Recursive quality loop

Beyond the initial ingestion of facts, the graph evolves through a recursive 
loop of human and autonomous feedback. This is known as **Intentional Agency**.

### Feedback as state

When a human or AI judge approves or corrects an agent's reasoning, that feedback 
is asserted as a verifiable triple in the world's ledger:

1.  **Capture preference**: The agent records a preference signal for a specific 
    query result or relationship.
2.  **Assert triple**: The system inserts mutation quads (e.g., 
    `<result_id> worlds:hasPreference worlds:Accepted`).
3.  **Refine state**: This historical truth remains in the ledger, allowing 
    future queries to follow the reshaped probability landscape.

---

## Probability reshaping

By providing a stateful substrate for feedback, Worlds enables the practical 
application of **Reinforcement Learning from Human Feedback (RLHF)** to steer 
model behavior.

### Logit shift and softmax filters

RLHF acts as a series of nudges in the model's token space:
*   **Logit shift**: Increases raw scores for preferred paths and suppresses 
    undesired ones.
*   **Softmax filter**: Ensures that during inference, the model is statistically 
    driven toward outcomes that align with the recorded preferences.

## Scalable supervision

As systems scale, human evaluation becomes a bottleneck. Worlds enables 
**Scalable Supervision** (RLAIF) by using high-reasoning models to evaluate 
candidate triples against a world model.

### Continuous evaluation (Evals)

Scalable Supervision is the operationalized form of an **Eval**.
*   **AI Judges**: Models evaluate candidate facts for consistency and safety.
*   **Autonomous Rewards**: Feedback is asserted autonomously (e.g., 
    `result:123 worlds:hasReward 0.95`), allowing the system to scale its 
    alignment without manual intervention.

---

## Constitutional AI

Worlds supports **Constitutional AI (CAI)**—alignment guided by a core set of 
axiomatic principles (the ontology) rather than manual labels.

*   **Self-Critique**: Agents use the graph to identify if their proposed 
    triples violate any constitutional rules.
*   **Collaborative Oversight**: Independent validator agents cross-verify 
    knowledge. Every approval is a reified fact (`worlds:verifiedBy :agent_A`), 
    creating a permanent, auditable log of the alignment process.

## Architecture

While agents interact with Worlds abstractly, the platform manages the 
underlying RDF data through a deterministic pipeline.

### RDF patching pipeline

1.  **Identification**: [Skolemizes](https://en.wikipedia.org/wiki/Skolem_normal_form) 
    and hashes each quad to a unique triple ID.
2.  **Relational Storage**: Upserts the resulting triples into the database.
3.  **Semantic Indexing**: Chunks and embeds literal values for 
    [hybrid search](/worlds/query).
4.  **Triggers**: SQL triggers synchronize `rdf:type` relations to specific 
    type tables.

### SPARQL execution

When executing a SPARQL query via the API, the system resolves the world-specific 
database, applies any pending updates, and executes the query over the 
resulting RDF blob using the **Comunica** engine.
