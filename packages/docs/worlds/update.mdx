An agent that can find and reason about data is useful, but an agent that can
actively _learn_ is autonomous. To continually improve a world, an agent must be
able to write new facts into the graph.

Worlds supports multiple strategies for ingesting new knowledge and continuously
updating data within an isolated world.

## SPARQL updates

At its core, a world mutates its graph state using RDF patches (granular
additions and deletions of facts). However, agents and developers rarely write
these patches manually.

The most common way to learn and mutate graph state is by executing SPARQL
updates. When an agent runs an `INSERT` or `DELETE` command, Worlds translates
the execution events into a deterministic RDF patch and applies it.

```sparql
PREFIX schema: <http://schema.org/>

INSERT DATA {
  <user:ethan> schema:worksAt <org:wazoo> .
}
```

## Ingestion strategies

### One-time import

Use the CLI or SDK to perform a bulk import of existing RDF data. This is ideal
for bootstrapping a new world with a base ontology or research dataset.

```bash
# Import a Turtle file into a world
worlds import --world <world-id> --file ./ontology.ttl
```

### Direct SDK ingestion

For real-time applications, use the `worlds-sdk` to insert individual triples or
chunks as they are generated by your agentic loops.

## Data synchronization

To keep your world memory seamlessly in sync with external operations, you will
typically utilize a forward-sync proxy to hook into **webhooks** exposed by your
services, such as repository events, team communication streams, or sales
pipelines.

1.  **Map external events**: Capture the real-time webhook payloads broadcast by
    your source system.
2.  **Translate to facts**: Convert the resulting JSON data into structured
    graph mutations (for example, translating a new repository ticket event into
    a SPARQL `INSERT` operation asserting that a specific user opened it).
3.  **Apply updates**: Execute the mutation against the world, allowing the
    engine to translate it into a safe, atomic RDF patch.

<Tip>
  Always [skolemize](https://en.wikipedia.org/wiki/Skolem_normal_form) blank
  nodes before ingestion to ensure deterministic triple IDs. Agents cannot
  reliably query or reason about unnamed nodes.
</Tip>
