The API Reference provides the definitive technical specifications for interacting with the Worlds engine programmatically. It is purposely segregated from high-level usage guides to focus purely on networking boundaries, raw inputs, and response structures.

## Core specifications

This documentation explicitly covers:

- **[REST Architecture](/reference/api)**: The foundational HTTP definitions, including base URLs and core resource paths.
- **[Authentication](/reference/api#authentication)**: Security protocols, header requirements, and API key management for Cloud and local instances.
- **[Rate Limits](/reference/api#rate-limits)**: Threshold policies, safe usage bounds, and standardized `429` error handling metrics.
- **OpenAPI Endpoints**: Fully interactive, auto-generated definitions mapping to every single available router endpoint.
