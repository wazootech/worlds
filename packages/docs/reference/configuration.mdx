# Configuration Reference

The Worlds platform switches between modes based on the presence of specific
environment variables.

## Management Console

These variables are required in `packages/console/.env` for production
deployments.

| Variable                       | Mode       | Purpose                                                              |
| :----------------------------- | :--------- | :------------------------------------------------------------------- |
| `WORKOS_API_KEY`               | Production | Enables Remote WorkOS Management.                                    |
| `WORKOS_CLIENT_ID`             | Production | Required for AuthKit login.                                          |
| `DENO_DEPLOY_TOKEN`            | Production | Enables remote Deno Deploy orchestration.                            |
| `TURSO_API_TOKEN`              | Production | Enables remote Turso database provisioning.                          |
| `TURSO_ORG`                    | Production | The Turso organization slug.                                         |
| `WORLDS_EMBEDDINGS_DIMENSIONS` | Both       | Unified vector dimensions for DB and AI (defaults to `768`).         |
| `OPENROUTER_API_KEY`           | Both       | Global fallback for embeddings.                                      |
| `OPENROUTER_MANAGEMENT_KEY`    | Production | Enables automatic per-app API key generation via OpenRouter API.     |
| `OPENROUTER_EMBEDDINGS_MODEL`  | Both       | Global fallback model (defaults to `openai/text-embedding-3-small`). |
| `OLLAMA_BASE_URL`              | Both       | Ollama API URL (defaults to `http://127.0.0.1:11434`).               |
| `OLLAMA_EMBEDDINGS_MODEL`      | Both       | Ollama model (defaults to `nomic-embed-text`).                       |

By default, the Worlds Platform uses **Ollama** for local embeddings to ensure
your data never leaves your machine.

> [!TIP] You can use **OpenRouter** for embeddings in local development by
> adding an `OPENROUTER_API_KEY` to your environment.

## World Server

These are automatically provisioned and injected by the Console during
deployment, but required if self-hosting the server manually.

| Variable                       | Purpose                                                                                 |
| :----------------------------- | :-------------------------------------------------------------------------------------- |
| `WORLDS_API_KEY`               | The **Root Admin Key** for the server. If omitted, the server runs in unprotected mode. |
| `WORLDS_EMBEDDINGS_DIMENSIONS` | Unified vector dimensions for DB and AI (defaults to `768`).                            |
| `LIBSQL_URL`                   | Connection string (e.g., `libsql://...` or `file:...`).                                 |
| `LIBSQL_AUTH_TOKEN`            | Auth token for remote libSQL/Turso instances.                                           |
| `OPENROUTER_API_KEY`           | For high-quality OpenRouter embeddings.                                                 |
| `OLLAMA_BASE_URL`              | Ollama API URL (defaults to `http://127.0.0.1:11434`).                                  |

## Embedding providers

The platform selects an embedding provider based on available environment
variables:

- **OpenRouter**: Triggered when `OPENROUTER_API_KEY` is present. Recommended
  for cloud-based AI.
- **Ollama**: Default provider when no cloud keys are present. Recommended for
  local development and private hosting.
