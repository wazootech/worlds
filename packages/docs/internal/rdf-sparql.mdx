# RDF & SPARQL

## RDF blob handling

- **lib/blob/n3.ts:** Utilities to convert between RDF blobs (e.g., N-Quads
  strings) and N3 Stores.
- **lib/blob/sparql.ts:** `sparql(blob, query, handler)` runs queries using
  Comunica over an N3 store.

## RDF patching

Centralized in `lib/rdf-patch/mod.ts`. The
`handlePatch(client, embeddings, patches)` function in `rdf-patch.ts` processes
mutations for the world database:

1. **Deterministic Identification**: Each quad is skolemized and hashed to a
   unique triple ID.
2. **Relational Storage**: Triples are upserted into the `triples` table.
3. **Semantic Indexing**: Literal values are chunked and embedded for hybrid
   search.
4. **Triggers**: SQL triggers sync `rdf:type` relations to the `entity_types`
   table.

## SPARQL route execution

1. Loads the world from the main DB.
2. Resolves the world client via `databaseManager.get(worldId)`.
3. Runs `handlePatch` with a handler that applies updates to the world client.
4. Executes the SPARQL query over the blob.
5. If the blob changed (on update), the main DB's world row is updated.
