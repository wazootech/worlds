# Resource Pattern

Each resource follows a consistent structure under
`packages/server/lib/database/core/` or `packages/server/lib/database/world/`:

- **queries.sql** — DDL and CRUD queries.
- **queries.sql.ts** — Generated string constants.
- **schema.ts** — Zod schemas.
- **service.ts** — Service class implementing methods.
- **service.test.ts** — Deno tests using in-memory client.

## Service mapping

| :--------------- | :------- | :-------------------------------------------------------- |
| chunks           | World    | search route, rdf-patch, testing                          |
| invites          | Main     | invites route, testing                                    |

| rate-limits      | Main     | middleware/rate-limit, init, testing                      |
