title: "Production Deployment"

---

To run a full production-grade instance of the Worlds Platform™, follow these steps:

### Provision Infrastructure

- **Identity (WorkOS)**:
  - Create a project in [WorkOS](https://workos.com) and set up AuthKit.
  - **Retrieve**: `WORKOS_API_KEY`, `WORKOS_CLIENT_ID`, and `WORKOS_COOKIE_PASSWORD`.
- **Compute (Deno Deploy)**:
  - Generate a personal access token in [Deno Deploy](https://dash.deno.com).
  - **Retrieve**: `DENO_DEPLOY_TOKEN`.
- **Storage (Turso)**:
  - Create a [Turso](https://turso.tech) account and generate an API token.
  - **Retrieve**: `TURSO_API_TOKEN` and `TURSO_ORG`.
- **AI Models (OpenRouter)**:
  - Create an account on [OpenRouter](https://openrouter.ai).
  - **Retrieve**: `OPENROUTER_API_KEY` (Global fallback).
  - **Management API (Advanced)**: Generate a Management API Key to enable automatic per-app key provisioning.
  - **Retrieve**: `OPENROUTER_MANAGEMENT_KEY`.

### Deploy the Console

The Console is a standard Next.js application. You can deploy it to Vercel, Netlify, or a VPS.

- Set the environment variables retrieved in **Step 1** in your deployment settings.
- Ensure `NEXT_PUBLIC_WORKOS_REDIRECT_URI` matches your deployment callback URL.

### Orchestrate Worlds

Once the Console is running:

1. Log in via AuthKit.
2. Create an **Organization**.
3. The platform will automatically:
   - Provision a new **Turso Database**.
   - Deploy a new **Deno Deploy App** running `packages/server`.
   - Link them together via metadata.

### Cloud Embeddings with OpenRouter

By default, the Worlds Platform™ is configured to use **OpenRouter** for cloud-based embeddings in production. This provides high-quality semantic search without requiring local GPU resources.

> [!TIP]
> You can customize the model and dimensions by setting `OPENROUTER_EMBEDDINGS_MODEL` and `WORLDS_EMBEDDINGS_DIMENSIONS` in your environment. See [Configuration Reference](/hosting/configuration) for details.

<Steps>
<Step title="Get an API Key">

Visit [OpenRouter](https://openrouter.ai/keys) to create your API key.

</Step>
<Step title="Configure the Console">

Add your key to the Console's environment variables:

```dotenv
OPENROUTER_API_KEY="your-api-key"
```

</Step>
<Step title="Automatic Injection">

The Console will automatically inject this key into every new World instance it provisions, enabling cloud embeddings by default.

</Step>
</Steps>
