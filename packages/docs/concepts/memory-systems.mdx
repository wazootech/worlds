# Memory Systems

In Worlds, memory is not a static log file—it is a live, evolving world state
that persists across sessions, platforms, and models.

## The Memory Hierarchy

### 1. Ephemeral Context

The volatile, short-term window of the current LLM prompt. This is limited by
tokens and is lost once the session ends unless materialized.

### 2. Persistent Entity Memory

Information extracted from interactions and stored in the Knowledge Graph. This
is the **Long-term Memory** of your agent, queryable via SPARQL or Hybrid
Search.

### 3. Edge Persistence

Worlds uses a localized storage model (powered by Turso and Deno), allowing
agents to retain memory even in high-privacy or disconnected environments.

## The Connector Strategy

In traditional systems, you "insert" data. In Worlds, you **connect** state. We
treat memory as a **continual learning loop** where agents self-improve through
constant exposure to new domain knowledge (GitHub, Slack, local docs).

### The Semantic Bridge

Connectors transform raw data into persistent memory in three steps:

1. **Extraction**: Pulling raw data from an external source.
2. **Triplification**: Mapping raw data to structured triples (facts).
3. **Resonance**: Pushing the triples into the persistent Knowledge Graph.
