# Knowledge Primitives & Items

To build applications with Worlds, you need to understand how the platform
stores the "reality" it extracts from your unstructured data. The Worlds engine
uses RDF triples as the fundamental primitive of knowledge in **Worlds**.

## The Item Drive Metaphor

To make these Semantic Web concepts intuitive, think of a World not as a
database, but as an **Item Drive**—the next evolution of the hard drive.

- **Items (The Nodes)**: An Item is synonymous with an **Entity**. These are the
  distinct "things"—a person, a piece of code, or an identity. They are the
  **Subjects** and **Objects** of your memory.
- **Properties (The Connectors)**: These are the "ports" or "labels" that define
  how Items relate (e.g., `worksOn`, `managerOf`). These are technically known
  as **Predicates**.
- **Facts (The Axioms)**: A **Fact** is created the moment you connect two Items
  using a Property. This connection is an **RDF Triple** (Subject → Predicate →
  Object)—the fundamental axiom of knowledge.

| Storage Era          | Primitive       | How it Works                                                 |
| :------------------- | :-------------- | :----------------------------------------------------------- |
| **Traditional Disk** | File / Folder   | Hierarchical silos: You must know the path to find the data. |
| **Item Drive**       | **Item / Fact** | Associative Graph: You follow the facts to find the items.   |

## 1. Worlds (The Container)

A **World** is an isolated container for a knowledge graph. It is the
fundamental unit of context and multi-tenant isolation.

- **Boundary:** Data in `World A` is strictly invisible to `World B`.
- **Use Cases:** You typically create one World per end-user for personal
  memory, or one World per team for organizational knowledge.
- **Components:** Behind the scenes, each World is an isolated namespace
  containing its own RDF Triplestore and Vector Index.

## 2. Entities (Nodes)

Entities are the subjects and objects in your World—the physical or conceptual
"nodes" in the graph. An entity can be a `User`, a `Task`, a `CodeSnippet`, a
`Repository`, or any abstract concept.

- **IRIs:** Every entity is identified by a unique Internationalized Resource
  Identifier (IRI).
- **Types:** Entities are categorized by their type (e.g., this entity is a
  `Company`, that entity is a `Person`).

## 3. Triples (Edges & Relationships)

A **Triple** is the smallest possible unit of knowledge, representing the
connections ("edges") between entities. A triple acts as a surgically precise
fact. It always consists of three parts:

1.  **Subject:** The entity being described.
2.  **Predicate:** The relationship verb.
3.  **Object:** The target value or another connected entity.

### Example

If an agent reads the sentence: _"Ethan is working on the API redesign."_

The Worlds engine extracts this as a Triple:

- **Subject:** `User:Ethan`
- **Predicate:** `isWorkingOn`
- **Object:** `Task:API_Redesign`

This structure ensures that the agent _knows_ this fact deterministically,
rather than just having a text chunk that happens to contain those words.

## Memory Types & Patterns

When designing your agent's memory, it is helpful to categorize information into
three functional types:

- **Facts (The Core)**: Stable, verifiable truths. These are long-lived and
  rarely change (e.g., birth dates, company names, project IRIs).
- **Preferences (The Vibe)**: Malleable traits that guide an agent's behavior.
  These evolve as the agent learns more about the user (e.g., "Prefers concise
  answers", "Likes TypeScript over Rust").
- **Episodes (The History)**: Event-based memories tied to specific moments or
  interactions (e.g., "The last time we discussed the API redesign", "The result
  of the system test on Friday").

By categorizing your triples into these patterns, you can build agents that are
not only smart but also deeply personalized.

## The Semantic Visualizer

Because Worlds are graph-based, they are fundamentally visual. Instead of
looking at JSON logs, you can literally "see" what your agent knows by viewing
its World.
