# Knowledge Primitives

The Worlds engine uses three fundamental primitives to structure knowledge for
AI agents: **Worlds**, **Entities**, and **Triples**.

## 1. Worlds (Units of Organization)

A **World** is an isolated container for knowledge. It is the fundamental unit
of multi-tenant isolation.

- **Isolation**: Data in one World is strictly invisible to others.
- **Components**: Each World manages its own RDF Triplestore and Vector Index.

## 2. Entities (Subjects and Objects)

Entities are the "things" in your World. They represent users, projects, code
snippets, or abstract concepts.

- **IRIs**: Every entity has a unique Internationalized Resource Identifier
  (IRI).
- **Types**: Entities are categorized via `rdf:type` (e.g., this entity is a
  `User`).

## 3. Triples (Surgical Truths)

The smallest unit of knowledge in the platform. A triple consists of a
**Subject**, **Predicate**, and **Object**.

- **Subject**: The entity being described.
- **Predicate**: The relationship (e.g., `hasDeadline`).
- **Object**: The value or another entity.

## The Semantic Visualizer

knowledge directly.
