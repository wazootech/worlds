# SPARQL Reasoning

While Vector Search is for "vibes," **SPARQL** is for "facts." It is the engine
that allows Worlds to provide deterministic precision in an era of probabilistic
AI.

## Deterministic vs. Probabilistic

LLMs often suffer from "semantic drift." If you ask a vector database for
"Project A's deadline," it might return a relevant paragraph about _Project B_
because they share similar keywords.

With SPARQL, the response is a binary truth. The fact either exists as a
relationship in the graph, or it doesn't.

```sparql
PREFIX w: <https://wazoo.dev/schema#>

SELECT ?deadline
WHERE {
  ?project w:name "Project A" ;
           w:hasDeadline ?deadline .
}
```

## Relational Fact Discovery

SPARQL excels at traversing the "Knowledge Graph" to find non-obvious links that
simple search cannot reach:

- **Logic Traversal**: "Find the direct manager of the person who committed this
  specific line of code."
- **Structural Integrity**: Ensuring that contradictory facts (e.g., two
  different birthdays for the same user) are flagged or resolved.
- **Inference**: Highlighting relationships that are implied but not explicitly
  stated (e.g., "If A is a child of B, and B is a sibling of C, then A is a
  niece/nephew of C").
